# Contributing to Cidadão.AI Technical Documentation

Thank you for your interest in contributing to Cidadão.AI's technical documentation! This guide will help you get started.

## How to Contribute

### Types of Contributions

- **Bug fixes**: Correct errors in documentation
- **Improvements**: Clarify existing documentation
- **New content**: Add missing documentation
- **Translations**: Help translate to other languages

### Getting Started

1. **Fork** the repository
2. **Clone** your fork locally
3. **Create a branch** (`git checkout -b docs/your-change`)
4. **Make changes** following our guidelines
5. **Commit** with clear messages
6. **Push** and create a Pull Request

### Documentation Guidelines

#### Structure

- Use clear, descriptive headings
- Organize content logically
- Include table of contents for long documents

#### Style

- Write in present tense
- Use active voice
- Be concise but complete
- Include practical examples

#### Code Examples

- Test all code examples before submitting
- Include comments explaining complex parts
- Use consistent formatting

### Building Locally

```bash
# Install dependencies
npm install

# Start development server
npm run dev

# Build for production
npm run build
```

## Review Process

1. All PRs require at least one review
2. Automated checks must pass
3. Documentation must render correctly
4. Changes should follow style guidelines
